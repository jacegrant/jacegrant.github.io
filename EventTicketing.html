Event Ticketing

Record Auditing system

Event Ticketing allows for organizations within the university to upload and transfer event tickets and parking passes to one another. After each transfer and upload, a log is created. These logs serve as audit logs allowing for each ticket or parking pass to be kept account of.

The page I created, shown below, lets an admin upload tickets or parking passes in bulk through the use of a csv file. The user selects the game(s), shown in the datatable, that they want to upload the records and is then taken through a popup "wizard" that instructs the user on each step of the process. Once the user is ready to upload the csv file of records, it is validated and the user is provided with the errors in the file and where they are, if any. When correct, the records are uploaded and put into the database for future use.

I utilized HTML, CSS, Bootstrap, and Javascript to create this view. The csv is parsed with the help of a library named Papaparse. The records are sent to the controller using Javascript, jQuery and Ajax and they are then handled with the help of C#, Linq, SQL and Entity Framework.
